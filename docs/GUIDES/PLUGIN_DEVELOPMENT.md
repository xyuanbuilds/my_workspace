# Plugin Development Guide

**Last Updated:** 2026-03-03

## Creating a New Plugin

### 1. Generate Scaffold

Use the create script to generate a new plugin:

```bash
# Create skill plugin
pnpm create-plugin my-tool --type skill

# Create hooks plugin
pnpm create-plugin my-hooks --type hooks

# Create MCP plugin
pnpm create-plugin my-mcp --type mcp
```

This creates the directory structure in `dev/my-tool/`.

### 2. Edit Source Files

**For Skill Plugin:**

Edit `dev/my-tool/src/index.ts`:

```typescript
export function run(input: string): string {
  // Implement your skill
  return `Processed: ${input}`;
}
```

Edit `dev/my-tool/src/cli.ts` for CLI interface.

Update `dev/my-tool/SKILL.md` with documentation.

**For Hooks Plugin:**

Edit `dev/my-hooks/hooks.json` to define hook matchers and handlers.

Edit `dev/my-hooks/scripts/example.js` with your hook logic.

**For MCP Plugin:**

Edit `dev/my-mcp/src/mcp-server.ts` to define tools and resources.

### 3. Add Dependencies

Edit `dev/my-tool/src/package.json`:

```json
{
  "dependencies": {
    "some-package": "^1.0.0"
  }
}
```

### 4. Build Plugin

Compile and package to `plugins/my-tool/`:

```bash
pnpm build:plugin my-tool
```

Build process:

1. ✓ TypeScript compilation (tsc)
2. ✓ Clean old output
3. ✓ Package files to plugins/ directory
4. ✓ Install production dependencies

### 5. Validate Structure

```bash
pnpm validate my-tool
```

Checks:

- `.claude-plugin/plugin.json` exists and is valid JSON
- `plugin.json` has required fields (name, version, etc.)
- Correct files for plugin type:
  - Skill: `skills/<name>/SKILL.md`
  - Hooks: `hooks/hooks.json`
  - MCP: `.mcp.json`
- No absolute paths in JSON files

### 6. Test Locally

Load the plugin in Claude Code:

```bash
pnpm test:plugin my-tool
```

This launches Claude Code with the plugin directory loaded.

### 7. Development Workflow

For iterative development, use watch mode:

```bash
# Terminal 1: Watch TypeScript compilation
pnpm dev my-tool

# Terminal 2: Load plugin in Claude Code
pnpm test:plugin my-tool
```

## Scaffold Structure

### Skill Plugin Template

```
dev/my-tool/
├── src/
│   ├── index.ts          # Main module
│   ├── cli.ts            # CLI entry point
│   └── package.json      # Runtime dependencies
├── dist/                 # Compiled JS (generated)
├── SKILL.md              # Skill documentation
├── plugin.json           # Plugin metadata
├── tsconfig.json         # TypeScript config
└── dist/                 # Output (generated by tsc)
```

### Hooks Plugin Template

```
dev/my-hooks/
├── scripts/
│   └── example.js        # Hook implementation
├── hooks.json            # Hook definitions
├── plugin.json           # Plugin metadata
└── tsconfig.json
```

### MCP Plugin Template

```
dev/my-mcp/
├── src/
│   ├── mcp-server.ts     # MCP server
│   └── package.json      # Runtime dependencies
├── dist/                 # Compiled JS (generated)
├── mcp.json              # MCP configuration
├── plugin.json           # Plugin metadata
├── tsconfig.json         # TypeScript config
└── package.json          # Dev dependencies
```

## Plugin Metadata (plugin.json)

Required fields in `plugin.json`:

```json
{
  "name": "my-plugin", // Unique name (alphanumeric, no spaces)
  "version": "1.0.0", // Semantic versioning
  "description": "What your plugin does", // Short description
  "author": {
    // Author info
    "name": "Your Name",
    "email": "email@example.com" // Optional
  },
  "keywords": ["tag1", "tag2"] // Search keywords (optional)
}
```

## Skill Documentation (SKILL.md)

Skills require a `SKILL.md` file with frontmatter:

````markdown
---
name: my-plugin
description: What this skill does
context: fork # execution context
allowed-tools: Bash(node *) # allowed tools
---

# My Plugin

Instructions for Claude on how to use this skill.

## Usage

```bash
node scripts/cli.js <arg>
```
````

```

## SKILL.md Frontmatter Fields

| Field | Required | Value | Notes |
|-------|----------|-------|-------|
| `name` | Yes | String | Must match plugin name |
| `description` | Yes | String | Brief description (one-liner) |
| `context` | No | `fork` \| `inline` | Execution context (default: inline) |
| `allowed-tools` | No | String | Tool restrictions (e.g., `Bash(node *)`) |

## Build Output Structure

After `pnpm build:plugin my-tool`, the plugin directory looks like:

```

plugins/my-tool/
├── .claude-plugin/
│ └── plugin.json # Copied from dev/my-tool/plugin.json
├── skills/my-tool/ # (skill type only)
│ ├── SKILL.md # Copied from dev/my-tool/SKILL.md
│ └── scripts/
│ ├── _.js # Compiled from dev/my-tool/dist/
│ ├── package.json # Copied from dev/my-tool/src/package.json
│ └── node_modules/ # pnpm install --prod output
├── hooks/ # (hooks type only)
│ └── hooks.json # Copied from dev/my-tool/hooks.json
├── scripts/ # (hooks type)
│ └── _.js # Copied from dev/my-tool/scripts/
└── .mcp.json # (mcp type or skill with mcp)

```

## Tips

- **Environment Variables in MCP:** Use `${CLAUDE_PLUGIN_ROOT}` for paths in .mcp.json
- **Dependencies:** Only production deps go in `src/package.json`. Dev deps stay in root.
- **Native Modules:** Be aware that `canvas` has native bindings. Users need compatible build tools.
- **Version Bumping:** Update `plugin.json` version before rebuilding for marketplace recognition.
- **Validation Before Publishing:** Always run `pnpm validate` before pushing.
```
